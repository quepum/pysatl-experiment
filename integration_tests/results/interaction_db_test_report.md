### Тест взаимодействия с базой данных (Critical Value)

**Цель проведения теста**:

Проверка стабильности и корректности работы параллельной реализации при интенсивном взаимодействии с базой данных SQLite, включая:
- параллельное чтение исходных данных из таблицы `random_values`,
- пакетную запись результатов в таблицу `limit_distributions`,
- отсутствие конфликтов и ошибок в условиях конкурентного доступа.

**Конфигурация**:

| Параметр                | Значение            |
|-------------------------|---------------------|
| Тип эксперимента        | `critical_value`    |
| Гипотеза                | `normal`            |
| Статистические критерии | KS, AD, SW, CVM, JB |
| Размеры выборок         | 200, 1000           |
| Уровни значимости       | 0.01, 0.05, 0.1     |
| Monte Carlo Count       | 1 000 итераций      |
| Общее число задач       | 5 × 2 × 3 = **30**  |
| Режим выполнения        | Параллельный        |

**Результаты выполнения**:

- **Статус завершения**: Успешно 
- **Ошибки СУБД**: Не выявлено
- **Число обработанных задач**: **30 из 30**

**Вывод**:

Все 30 задач завершились успешно, результаты корректно сохранены в таблице `limit_distributions`. Отсутствие ошибок подтверждает корректность принятой архитектуры:
- **Чтение исходных данных** выполняется в изолированных процессах-воркерах (режим «только чтение»),
- **Запись результатов** осуществляется в основном потоке с использованием буферизации, что исключает конкурентный доступ к записи.

Несмотря на высокую вариативность длительности задач, система продемонстрировала устойчивую работу:
- Длительные задачи (AD, SW на больших выборках) выполнялись параллельно с краткосрочными
