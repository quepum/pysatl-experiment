### Тест устойчивости к сбоям (Time Complexity)

**Цель проведения теста**:  
Проверка способности системы к сохранению результатов успешно завершённых задач при возникновении ошибки в одной из задач.

**Конфигурация**:  

| Параметр                | Значение |
|-------------------------|---|
| Тип эксперимента        | `time_complexity` |
| Гипотеза                | `normal` |
| Статистические критерии | JB |
| Размеры выборок         | 100, 200, 300, 500, 1000 |
| Уровни значимости       | — |
| Альтернативы            | — |
| Monte Carlo Count       | 1 000 |
| Общее число задач       | 5 |
| Режим выполнения        | Параллельный |
| Имитация сбоя           | Задача `JB`, `sample_size=500` |

**Результаты выполнения**:  

- **Статус завершения**: Успешно  
- **Общее время выполнения**: **12.74 секунд**  
- **Пиковое потребление CPU**: **18.2%**  
- **Ошибки СУБД**: Не выявлено  
- **Число обработанных задач**: **4 из 5**  
- **Время выполнения отдельных задач**:  
  - Минимальное: **0.8 с** (JB, n=100),  
  - Максимальное: **5.1 с** (JB, n=1000).

**Вывод**:  
Сбой в одной задаче (`JB`, `n=500`) не прерывает выполнение остальных. Все 4 успешно завершённые задачи гарантированно сохранены в таблице `time_complexity`, а execution-шаг завершается со статусом «успех».
