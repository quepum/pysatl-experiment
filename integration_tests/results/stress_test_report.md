### Стресс-тест параллельной реализации (Power-эксперимент)

**Цель проведения теста**:  
Проверка стабильности, масштабируемости и производительности параллельной реализации при обработке большого числа разнородных задач (90 задач) в условиях длительного выполнения.

**Конфигурация**:  

| Параметр                | Значение |
|-------------------------|----|
| Тип эксперимента        | `power` |
| Гипотеза                | `normal` |
| Статистические критерии | KS, AD, SW |
| Размеры выборок         | 100, 200, 300, 400, 500 |
| Significance levels     | 0.01, 0.05, 0.1 |
| Альтернативы            | Gamma(0.5, 1.0), Weibull(1.0, 2.0) |
| Monte Carlo Count       | 1 000 |
| Общее число задач       | 90 |
| Режим выполнения        | Параллельный |

**Результаты выполнения**:  

- **Статус завершения**: Успешно  
- **Общее время выполнения**: **482.01 сек**  
- **Пиковое потребление CPU**: **20.4%**  
- **Ошибки СУБД**: Не выявлено  
- **Число обработанных задач**: **90 из 90**  
- **Время выполнения отдельных задач**:  
  - Минимальное: **3.2 с** (KS, n=100),  
  - Максимальное: **18.7 с** (AD, n=500).

**Вывод**:  

Система стабильно обработала 90 независимых задач без единой ошибки. Равномерная и умеренная загрузка CPU (12–20%) свидетельствует об эффективном распределении нагрузки и отсутствии перегрузки. Все результаты успешно сохранены в таблице благодаря буферизованной записи в основном потоке.
